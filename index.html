Almacenamiento de Información

Los navegadores actuales están en un punto, en el cual se pueden tener aplicaciones completamente independientes sin necesidad de una API, solo con cargar nuestro frontend y tener todos los datos de manera estática. Incluso podemos tener sitios web que sin estar conectados a internet funcionen.

Los navegadores guardan los datos de las páginas en que navegamos, y la manera en que guardan dicha información están divididas en 3 fundamentales métodos:

01

SessionStorage

El sessionStorage es posiblemente el menos utilizado de los métodos para almacenar información en una aplicación web.

02

Local Storage

El localStorage es muy similar al sessionStorage, con la única diferencia que este puede ser compartido entre varias ventanas del navegador y perdura en el tiempo.

03

IndexedDB

El caso del indexedDB es completamente diferente a los métodos anteriores, dado que cómo su nombre lo indica, es una base de datos.

Los usuarios indican a las computadoras que accedan a los datos y los almacenen en estos dispositivos de almacenamiento.

Sin embargo, a un nivel fundamental, hay dos bases para el almacenamiento de datos: la forma en que toman los datos y los datos de los dispositivos en los que se registran y almacenan.

SessionStorage

Es una propiedad que acceden al objeto Storage y tiene la función de almacenar datos de manera local, esta almacena información mientras la pestaña donde se esté utilizando siga abierta, la sesión de la página se mantiene por sobre las recargas. Una vez cerrada la pestaña, la información se elimina.

Caracteristicas del SessionStorage:

  • El sessionStorage objeto almacena datos solo para una sesión: Esto quiere decir que los datos almacenados en el sessionStorage se eliminarán cuando se cierre el navegador.
  • La sesión existe mientras el navegador esté abierto: la sesión existe mientras el navegador web esté abierto y sobreviva a la actualización de la página.
  • SessionStorage separada: Esto sucede cuando se abren varias pestañas o ventanas con la misma URL, el navegador web crea un sessionStorage separada para cada pestaña o ventana. Por lo tanto, no se puede acceder a los datos almacenados en una pestaña del navegador web en otra pestaña.
  • Cuando cierra una pestaña o ventana, el navegador web finaliza la sesión y borra los datos del archivo sessionStorage.
  • Los datos almacenados en el sessionStorageson específicos del protocolo de la página: un mismo sitio javascripttutorial.nettiene diferentes sessionStorageal acceder con las teclas httpy https.

Metodos

setItem():

Guarda la información valor a la que se podrá acceder invocando a clave. Por ejemplo clave puede ser "nombre" y valor puede ser "Josuu".

sessionStorage.setItem(‘nombre’, ‘Josuu’);

getitem():

Recupera el value de la clave especificada.

sessionStorage.getItem(‘nombre’);

Por ejemplo si clave es nombre puede recuperar “Josuu”.

length:

Devuelve el número de items guardados por el objeto sessionStorage actual

sessionStorage.length;

key():

Cada item se almacena con un índice que comienza por cero y se incrementa unitariamente por cada item añadido. Con esta sintaxis rescatamos la clave correspondiente al item con índice i.

sessionStorage.key(i);

removeItem():

Elimina un item almacenado en sessionStorage.

sessionStorage.removeItem();

clear():

Elimina todos los items almacenados en sessionStorage, quedando vacío el espacio de almacenamiento.

sessionStorage.clear();

LocalStorage

La propiedad LocalStorage que nos permite acceder al objeto localStorage, el cual permite a JavaScript, a los sitios web y aplicaciones web, a almacenar valores e índices, o información en el navegador web, la mayor diferencia con la SessionStorage, es que esta no tiene fecha de caducidad, a comparación de la SessionStorage, que se mantienen los datos mientras la sesión del navegador esté activa.

Al menos en el caso del navegador Google Chrome, la información se guarda en un archivo SQLite, en una subcarpeta del perfil del usuario, la carpeta está localizada en /AppData/Local/Google/Chrome/UserData/Default/Local Storage, para el navegador Firefox se guarda en un archivo SQLite llamado webappsstore.sqlite.

Tendremos acceso al LocalStorage por medio de la propiedad de WIndow.localStorage, este es parte de la interfaz de JS de Window, que representa una ventana que contiene un documento DOM. Es de considerar que esta solo es una propiedad de lectura que retorna la referencia al objeto del localstorage usado para almacenar los datos que solo son accesibles por el origen.

Metodos

setItem():

Este método nos permite, guardar valores en el objeto de localStorage, para hacerlo necesitamos dos parámetros, el índice y el valor, de ejemplo tenemos lo siguiente:

window.localStorage.setItem(‘nombre’, ‘Kennet Vargas’);

Donde “nombre” es el indice y “Kennet Vargas” es el valor, notese que el localStorage solo es capaz de almacenar datos de tipo String, para poder almacenar arreglos u objetos, tendremos que convertirlos a strings.

getitem():

El siguiente método permite accede a la información almacenada en el objeto localStorage del navegador del usuario, este solamente acepta un parámetro y este es el índice, posterior retorna el valor de este índice como un dato de tipo string. Un ejemplo sencillo es:

window.localStorage.getItem(‘nombre’);

El valor a retornar sería de “Kennet Vargas”.

getitem():

El siguiente método permite accede a la información almacenada en el objeto localStorage del navegador del usuario, este solamente acepta un parámetro y este es el índice, posterior retorna el valor de este índice como un dato de tipo string. Un ejemplo sencillo es:

window.localStorage.getItem(‘nombre’);

El valor a retornar sería de “Kennet Vargas”.

removeItem():

Para eliminar datos almacenados en la sesión del localStorage, usa el método de removeItem(), cuando se pasa un índice, se eliminan todos los datos relacionados a este índice si este existe, de lo contrario no hará nada.La ejemplificación básica es:

window.localStorage.removeItem(‘nombre’);

clear():

Para eliminar todos los datos y elementos que hay en el localStorage, se usa el método de clear(). este no recibe ningún parámetro.

window.localStorage.clear();

key():

Este método resulta útil en momentos donde necesitamos recorrer los índices y nos permite retornar un número o índice al localStorage para recuperar el nombre de la clave.

var Indice = window.localStorage.key(indice);

Algunas desventajas que podemos encontrar del localStorage son:

  • No almacenar información privada o sensible del usuarios.
  • No es un sustituto para una base de datos.
  • Está limitado a 5MB de almacenamiento en la mayoría de los navegadores.
  • No es seguro, ya que no tiene ninguna clase de seguridad para proteger los datos.
  • Es sincrónico, significa que cada operación llamada, será ejecutada una después de la otra

IndexedDB

IndexedDB es una API de bajo nivel para el almacenamiento del lado del cliente de cantidades significativas de datos estructurados, incluidos archivos/blobs.

Esta API utiliza índices para permitir búsquedas de alto rendimiento de estos datos. En otras palabras, es una base de datos construida dentro del navegador, mucho más potente que localStorage. Los datos son guardados bajo el directorio raíz del usuario junto con la configuración personal del navegador, extensiones, etc.

Caracteristicas del SessionStorage:

  • Gran Capacidad de Almacenamiento.
  • Restricción de Dominio.
  • Soporta Transacciones
  • Es Asíncrona
  • Almacena pares clave/valor
